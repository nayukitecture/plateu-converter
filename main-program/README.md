# PLATEAU CityGML → OBJ 変換ツール

## 概要

国土交通省 PLATEAU が提供する CityGML データを OBJ 形式に変換する Python スクリプトです。  
ブロック（区画）単位で 1 つの OBJ ファイルを生成し、Rhinoceros などの 3D ソフトに直接読み込めます。  
複数ブロックを指定した場合は 1 つのファイルにまとめて出力します。

---

## データのダウンロード

PLATEAU オープンデータは以下から入手できます。

**https://www.mlit.go.jp/plateau/open-data/**

ダウンロードした ZIP を展開し、`resources/input/` 以下に配置してください。  
`resources/input/` フォルダがない場合は初回起動時に自動で作成されます。

```
resources/input/
  13209_machida-shi_pref_2023_citygml_2_op/    ← 展開したフォルダをここに置く
  14100_yokohama-shi_city_2024_citygml_2_op/
```

`resources/input/` に置いておくと対話モードで番号選択できて便利ですが、別の場所にあるフォルダもパスで直接指定できます。

サンプルデータとして `13209_machida-shi_pref_2023_citygml_2_op` の一部ブロックを同梱しています。  
そのまま実行して動作確認できます。

---

## 必要環境

- Python 3.8 以上
- 追加ライブラリ不要（標準ライブラリのみ使用）

---

## 使い方

`main-program/` フォルダで実行します。

### 通常の使い方（対話モード）

引数なしで実行すると、順に質問されて入力する方式で進みます。

```bash
python plateu-converter.py
```

実行すると以下の順に入力を求められます。

```
============================================
  PLATEAU CityGML → OBJ
　変換ツールを開始します。
============================================
【入力フォルダ (-i)】
  利用可能なデータセット:
    1: 13209_machida-shi_pref_2023_citygml_2_op
    2: 14100_yokohama-shi_city_2024_citygml_2_op
  番号で選択できます。リスト以外の場所にあるフォルダはパスを直接入力してください。
  入力フォルダ: 

【ブロック番号 (-b)】
  ※ ブロック番号は GML ファイル名の先頭の数字です（国土地理院の地図メッシュ番号）。
  例: 53392307 53392308 53392339
  ブロック番号: 

【オブジェクト種別 (-t)】
  選択可能な種別:
    bldg    建物
    frn     都市設備
  種別: 

【LOD レベル (bldg: 建物)】
  利用可能なLOD: 1/2
  LOD [1/2] (既定: 2): 

【マテリアルモード (-m)】
  1: なし          マテリアルなし（最も軽量）
  2: 部位別カラー  屋根・壁・地面などを色分け（MTL ファイルを追加出力）
  3: フォトテクスチャ  撮影写真を貼り付け（※ データ量が大幅に増加）
  マテリアルモード [1/2/3] (既定: 1): 

【出力フォルダ名 (-o)】
  出力フォルダ名 (既定: 日時自動): 
```

| 質問 | 入力内容 | 備考 |
|------|---------|------|
| 入力フォルダ | 番号、またはフォルダのパス | `resources/input/` 内のフォルダは番号で選択可。それ以外の場所はパスを直接入力 |
| ブロック番号 | 数字（スペース区切りで複数可） | GML ファイル名の先頭の数字（国土地理院の地図メッシュ番号。3次メッシュ=6桁・4次メッシュ=8桁など）。入力フォルダ選択後に例が表示される。プラトーDLデータに含まれるPDFも参照 |
| オブジェクト種別 | 種別コード（スペース区切りで複数可） | 入力データに存在する種別が一覧表示される |
| LOD レベル | 表示された候補から選択 | 種別ごとに個別に聞かれる。GML を走査して実際に存在する LOD のみ提示される |
| マテリアルモード | `1` / `2` / `3` | 下記「マテリアルモード」を参照 |
| 出力フォルダ名 | `resources/output/` 内のサブフォルダ名 | 省略すると日時フォルダ（`yyyymmdd-hhmmss`）を自動生成 |

すべての質問に答えると変換が始まります。変換完了後には、同じ設定で再実行するためのコマンドライン引数が表示されます。

```
【同じ設定で再実行する場合のコマンド】
  python plateu-converter.py -i "..." -b 53392339 -t bldg -lod 2 -m 2 -o machida
```

### コマンドライン引数で実行する方法

引数を直接指定して実行することもできます。

```bash
python plateu-converter.py -t bldg -lod 1 -b 53392339 -o sample -i "../resources/input/13209_machida-shi_pref_2023_citygml_2_op"
```

部位別カラー付きで変換する場合は `-m 2` を追加します。

```bash
python plateu-converter.py -t bldg -lod 2 -m 2 -b 53392339 -o machida -i "../resources/input/13209_machida-shi_pref_2023_citygml_2_op"
```

複数の種別をタイプ別に異なる LOD で変換する場合は、`-t` と `-lod` を同数指定します（位置対応）。

```bash
# bldg=LOD2、frn=LOD3 で変換
python plateu-converter.py -t bldg frn -lod 2 3 -b 51357380 51357370 -m 2 -o osaka-expo -i "../resources/input/27999_osaka-shi_city_2025_citygml_1_op"
```

```bash
# 全種別を同じ LOD で変換（-lod を 1 つだけ指定）
python plateu-converter.py -t bldg frn -lod 2 -b 51357380 -o osaka -i "../resources/input/27999_osaka-shi_city_2025_citygml_1_op"
```

## オプション一覧

| オプション | 取りうる値 | 説明 | デフォルト |
|-----------|-----------|------|-----------|
| `-i <DIR>` | フォルダパス | 入力データセットのルートフォルダ（`udx/<type>` まで自動探索） | （必須） |
| `-t <TYPE...>` | 下記の種別コード（スペース区切りで複数可） | オブジェクト種別。複数指定すると種別ごとに別ファイルで出力 | `bldg` |
| `-lod <N...>` | `1` / `2` / `3` / `4`（スペース区切りで複数可） | LOD レベル。`-t` と同数指定するとタイプ別に適用。1 つだけ指定した場合は全タイプに適用 | `1` |
| `-m <N>` | `1` / `2` / `3` | マテリアルモード（下記参照） | `1` |
| `-b <BLOCK...>` | 数字のブロック番号（スペース区切りで複数可） | 変換するブロック番号。省略時は全ブロック | （全件） |
| `-o <NAME>` | 任意の名前（例: `yokohama`, `machida-station`） | `resources/output/` 内に作成するフォルダ名。省略時は日時フォルダを自動生成 | 日時自動 |

### 対応オブジェクト種別

| 種別コード | 内容 |
|-----------|------|
| `bldg` | 建物 |
| `brid` | 橋梁 |
| `tran` | 交通（道路・鉄道） |
| `luse` | 土地利用 |
| `veg`  | 植生 |
| `frn`  | 都市設備 |
| `fld`  | 洪水浸水想定区域 |
| `htd`  | 高潮浸水想定区域 |
| `tnm`  | 津波浸水想定区域 |
| `lsld` | 土砂災害危険区域 |
| `urf`  | 都市計画区域等 |
| `ubld` | 地下建物 |

## マテリアルモード

`-m` オプションで OBJ に付加するマテリアル情報を選択できます。

| モード | 名称 | 内容 |
|--------|------|------|
| `1` | なし | マテリアルなし。OBJ のみ出力（最も軽量） |
| `2` | 部位別カラー | 屋根・壁・地面などのサーフェス種別ごとに色を割り当て。OBJ と MTL を出力 |
| `3` | フォトテクスチャ | PLATEAU に同梱の撮影写真 JPG を UV マッピング。OBJ と MTL を出力。**（※ データ量が大幅に増加）** |

### 部位別カラー（-m 2）の色割り当て

| サーフェス種別 | 色 |
|---|---|
| RoofSurface（屋根） | テラコッタ（赤橙） |
| WallSurface（外壁） | コーンフラワーブルー |
| GroundSurface（地面） | オリーブグリーン |
| OuterCeilingSurface（外部天井） | ティール |
| OuterFloorSurface（外部床） | パープル |
| ClosureSurface | ライムグリーン |
| InteriorWallSurface（内壁） | サーモン |
| FloorSurface（室内床） | アンバー |
| CeilingSurface（室内天井） | ラベンダー |
| Door（扉） | シアン |
| Window（窓） | オーカー |
| LOD1代替・未分類 | ライトタン（デフォルト） |
| 上記以外の未知種別 | 黄金比 HSV で自動生成（重複なし） |

### フォトテクスチャ（-m 3）の注意

- フォトテクスチャは LOD2 以上の建物データ（`bldg`）に同梱されている撮影写真 JPG を使用します
- JPG ファイルは入力フォルダ内の `_appearance/` サブフォルダに格納されています
- OBJ ファイルは JPG への絶対パスで参照するため、**OBJ・MTL だけ別フォルダに移動するとテクスチャが切れます**。別 PC や別フォルダに移す場合は JPG（`_appearance/` フォルダ）も一緒にコピーしてください

## 出力

- **出力先**: `resources/output/<フォルダ名>/`
- **ファイル構成**:
  - OBJ ファイル: `{フォルダ名}_LOD{N}_{type}_{ブロック番号...}.obj`
  - MTL ファイル: 同名の `.mtl`（`-m 2` または `-m 3` 指定時のみ）
  - 例: `machida-station_LOD2_bldg_53392345_53392355.obj` / `.mtl`
- **内部構造**: オブジェクトごとに `o {ID}` で区切られる  
  → Rhinoceros で開いて **Explode** すれば 1 つずつのオブジェクトになります

### Rhinoceros への取り込み

OBJ ファイルをそのまま Rhino にドラッグ＆ドロップするか、`File > Import` で読み込みます。  
**OBJ と MTL は同じフォルダに置いたまま**にしてください。OBJ をインポートすると MTL が自動で参照され、マテリアルが適用されます。

- **部位別カラー（-m 2）**: マテリアルパネルに `mat_RoofSurface`、`mat_WallSurface` などが自動登録されます
- **フォトテクスチャ（-m 3）**: マテリアルパネルに JPG テクスチャが貼り付けられた状態で登録されます

### 完了時の表示例

**LOD1 の場合**

```
======= bldg / LOD1 / マテリアル: なし / 1 ブロック =======
  53392339_bldg_6697_op.gml: 12 棟
  総数     : 12 棟
========================================

  出力先   : .../resources/output/sample/
　変換が完了しました。
```

**LOD2・部位別カラーの場合（複数タイプ・複数ブロック）**

```
======= bldg / LOD2 / マテリアル: 部位別カラー / 2 ブロック =======
  53392345_bldg_6697_op.gml: 425 棟  （LOD1代替: 178）
  53392355_bldg_6697_op.gml: 1819 棟  （LOD1代替: 634）
  総数     : 2244 棟
  LOD2     : 1432 棟
  LOD1代替  : 812 棟
========================================
```

## 座標系

等距離円筒図法（基準緯度 35°）でミリメートル変換して出力します。Rhino の標準単位（mm）に合わせています。

| 軸 | 方向 |
|----|------|
| X | 東（経度方向） |
| Y | 北（緯度方向） |
| Z | 高さ（mm） |

異なる都市・地区のデータも同じ変換式を使うため、そのまま合体できます。

## LOD について

| LOD | 形状の詳細度 | 主な対象種別 |
|-----|-------------|-------------|
| LOD1 | フットプリントを押し出したボックス形状 | bldg など |
| LOD2 | 屋根形状あり（切妻・陸屋根など） | bldg など |
| LOD3 | 開口部・細部形状あり | frn（都市設備）など |
| LOD4 | 室内・内部構造あり | 種別・データセットによる |

利用可能な LOD は種別・データセットによって異なります。対話モードでは GML を走査して実際に存在する LOD のみ選択肢として提示します。

指定した LOD のデータがないオブジェクトは、自動的に下位 LOD（LOD{N-1} → LOD{N-2} → … → LOD1）で代替します。変換完了時にどの LOD が何件使われたか内訳を表示します。
